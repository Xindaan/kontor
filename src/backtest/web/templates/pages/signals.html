Workflow right here
1. In the Strategy dropdown above, select the strategy you are actually currently holding.
2. Under Candidates, add only the real alternatives that a switch should be checked against.
3. Run Signals and only switch in the report when switch_allowed=true and the evidence is PASS.
When do I use Meta Decision vs. Meta Bootstrap?
Meta Bootstrap answers the starting question: which strategy do I start with, neutrally?
Meta Decision answers the follow-up question: am I allowed to switch away from my current strategy today?
Recommended defaults are already active here: hard evidence, profile ausgewogen, regime mode strategy_fragility, fail action hold_current.

Expected JSON: an array of strings or objects with strategy and optional params.

Example: [{"strategy":".../sticky_winner.py","params":{"momentum_floor":0.0}}]

Compares both directions (A->B, B->A) and provides a neutral starting recommendation with evidence + fallback rule.

Workflow / When do I use this?
1. Select two strategies you want to decide between initially.
2. If a strategy has a short history, shorten the bootstrap windows below.
3. Click Run Start Decision and adopt the recommended starting strategy.
4. After that, only check ongoing switches against the currently held strategy in the Meta Decision block above.
Bootstrap Window
Default for long histories: 5Y train / 1Y test / 12M step. Shorter for Maxblue or newer products.
